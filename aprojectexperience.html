  • 7500+ Dollars Saved

Project Description

SEMI, a global trade association of Semiconductor companies, seeks to establish a comprehensive dashboard to aggregate and analyze water and waste data reported by its member companies. This project can be broken down into three steps: data retrieval, data extraction, and data presentation. This project aims to collect environmental data in the form of sustainability reports, consolidate that data, and present it for SEMI’s major stakeholders.

Project Proposal

  • Solution: The data extraction program will have the automated capability to retrieve PDF links, extract relevant fields from the PDFs, and output the relevant data. The project will deliver an interactive, intuitive PowerBI dashboard with capabilities including real-time updates, advanced analytic visualizations, and benchmark reporting. The project will produce complete documentation of the project development and final results, as well as a final presentation for project demonstration and handoff.
  • Tech Stack: HTML, CSS, JavaScript, Python, Flask, AWS S3 Bucket via Heroku Bucketeer

Functional Scope

  • Develop a web-scraping bot that will grab the sustainability reports from each of the companies specified by project sponsor
  • Develop a method for extracting information from these reports
  • Develop a dashboard/analytics visualization

Non-Functional Scope

  • Perform visualizations on data for major stakeholders to view and pull insights from; since it is a trade association, it can be used to set environmental guidelines
  • Make sure web-scraping bot pulls most updated information so that analytics are accurate for stakeholders
  • Ensure PDF scanner extracts most accurate waste/water usage

Restaurant Reservation Manager

Project Description

This was the final project for my Object Oriented Design Class at San Jose State University. Creating objects, classes, methods, and inheriting/extending classes/interfaces with principles such as polymorphism, encapsulation, and multithreading were all considered. Working collaboratively as an Individual Contributor, making pull requests, and managing merge conflicts were all experiences I was glad to have with this project.

Project Proposal

  • Problem: Nowadays, making phone calls and manually keeping track in a notebook are the only ways to make bookings. When this happens, clients may have to wait to make reservations over the phone since it is busy. It may also be expensive and time-consuming to manually maintain track. Additionally, it takes time for consumers to phone the restaurant again in order to cancel their reservation.
  • Vision: Create a seamless booking experience on the web for a private sushi pop-up.
  • Domain: The target audience would include the owner of the sushi pop-up, who wants to automate the booking process, as well as the customers who want to conveniently and save time when making a reservation.
  • Tech Stack: Java, Java Swing

Functionality

Users will be able to book reservations through a window, view the menu, and cancel reservations. The program should also keep track of booked reservations and make updates to reservations.

High-level Description

Our plan is to create a reservation site that gives users the ability to create reservations at any local sushi restaurant without calling it in as well as managing the reservation for users who have a restaurant of their own. Specifically, users can use this site to choose the restaurant they would like to dine at and fill in all the needed information such as the time and size of their party. As for the owner, they can keep track of their customer’s name, phone number and the reserved table automatically just by opening up the site. Lastly, If the customers decide to cancel their reservation, they can simply do so through the site instead of calling the restaurant.

Final Result:

After presenting our project to the class, our professor was impressed with the funcationality of our program. He was not expected the admin page to be able to view the reservation Objects created. My team and I thought long and hard about the user experience of this application, so we made sure to put in things like error messages to simulate what would happen to someone actually using this. QuickCash

Description

  • Proposal: After my time at Cisco, I wanted to try my hand on a personal project that I would actually use in real life. Whenever I file my taxes, I always wanted a quick and easy way to quickly compare the amount that I actually owe next to a rough estimate. Using what I know about the individual income tax laws: post/pre tax deductions, varying state income tax rates, and filing statuses, I sought out to create a simple tool in to acheive this goal. I had envisioned a simple layout where users can input a limited amount of information and be returned with the amount of taxes that they owe.
  • Tech Stack:H TML, CSS, JavaScript, Node.js, Express.js

Functionality

When using the site, it's extremely simple. At the top, there is a navigation bar that takes you to either individual income taxes or a wage to salary converter. On both of these pages, the left hand side is where you input your personal information and the left hand side is where you see the results.
